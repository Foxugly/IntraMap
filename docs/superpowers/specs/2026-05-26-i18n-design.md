# Spec — Internationalisation (FR + EN) du GUI

Date : 2026-05-26
Statut : validé (GO — niveau « complet »)

## Objectif

Rendre le **chrome** de l'interface bilingue (français source + anglais),
avec un choix de langue persistant.

## Mécanisme — `intramap/gui/i18n.py`

Catalogue FR→EN explicite (pas de `.ts/.qm` Qt : testable sans outillage,
rien à compiler en CI).

```python
def tr(text: str) -> str         # source FR si langue=fr, sinon lookup catalogue (repli = source)
def set_language(lang)           # "fr" | "en"
def current_language() -> str
def available_languages()        # [("system","Système"),("fr","Français"),("en","English")]
def resolve_system_language()    # locale -> "fr"/"en"
def load_saved_language()        # QSettings "language" (défaut "system")
def save_language(choice)        # QSettings
def apply_saved_language()       # set_language selon la préférence (résout "system")
```

- `tr` est pur (lookup dict) : testable sans QApplication.
- Catalogue : `_CATALOG["en"][<source FR>] = <anglais>` ; une chaîne absente
  retombe sur le français (jamais d'exception).

## Périmètre des chaînes enrobées

Tout le chrome GUI : titres de fenêtres/dialogues, actions/menus, boutons,
labels de formulaires, placeholders, messages de barre de statut, en-têtes,
tooltips. **Hors périmètre** : contenu des rapports
(`wiring_report`/`path_report`/`diagnostics`/`scan_diff`, partagés avec le CLI)
et messages CLI.

## UI — choix de la langue

Menu **Affichage → Langue** (groupe exclusif : Système / Français / English,
coché selon la préférence). Sélection → `save_language` + message
« La langue sera appliquée au prochain démarrage. » (pas de retraduction live).

## Démarrage — `intramap/gui/app.py`

`i18n.apply_saved_language()` avant la création de `MainWindow`.

## Tests — `tests/test_i18n.py`

- `tr` : langue fr → renvoie la source ; langue en → traduction du catalogue ;
  chaîne inconnue en en → repli sur la source.
- `set_language` invalide → fr.
- `resolve_system_language` mappe `fr_*` → "fr", sinon "en" (monkeypatch
  locale).
- **Complétude** : pour chaque chaîne enrobée recensée (voir plus bas), le
  catalogue EN possède une entrée → garantit qu'il n'y a pas de FR résiduel
  en anglais. Mécanique : un test parcourt un ensemble de chaînes-clés
  attendues et vérifie leur présence dans `_CATALOG["en"]`.

## Exécution par lots

1. Infrastructure (`i18n.py` + tests + menu Langue + `app.py`).
2. Enrobage + catalogue EN, fichier par fichier
   (`main_window`, dialogues, `canvas`, `node`, …), un commit par lot, suite
   verte à chaque étape.

- Créer : `intramap/gui/i18n.py`, `tests/test_i18n.py`.
- Modifier (lots) : tous les modules `intramap/gui/*.py` portant des chaînes
